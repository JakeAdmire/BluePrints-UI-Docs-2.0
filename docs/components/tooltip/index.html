Tooltip

Tooltips display helpful information in convenient places
warning

Documentation is being worked on; some pages may be incomplete or not yet exist!

View source for this component on GitHub

Tooltips provide additional context to an element. We make it easy for you by providing a quick & easy Tooltip component. Just wrap it around any element you need and it's ready to go.

import { Tooltip, Button } from 'blueprints-ui';
return (
    <Tooltip text="I am a tooltip">
        <Button backgroundColor="#0097e0" 
                text="hover over me" />
    </Tooltip>
)
I am a tooltip

#Attributes

Options that change the look of your component

#text

Type: string|HTMLElement | Default: null

The Tooltip is intended to provide additional context an element of your choosing. Use the text attribute to fill in that context with a string or html.

Try not to get too fancy with html within the Tooltip, it's intended for basic syntax improvements only.

<Tooltip text="I'm a regular string">
    <Button />
</Tooltip>
<Tooltip text={<span>I'm made with <i>fancy</i> <b>HTML</b></span>}>
    <Button />
</Tooltip>
I'm a regular stringI'm made with fancy HTML

#pointed

Type: boolean | Default: true

Sometimes you need a tooltip with some direction, sometimes you don't. The pointed attribute allows you to customize whether or not the Tooltip has an arrow pointing towards the child element.

<Tooltip> // You can also use `pointed` or `pointed={true}`
    <Button />
</Tooltip>
<Tooltip pointed={false}>
    <Button />
</Tooltip>
I have a pointI'm pointless

#position

Type: string | Default: "bottom"

The Tooltip can exist on any of the four sides of it's child element:

  • top
  • right
  • bottom
  • left

Each of these positions also has a start & end alignment, totaling 12 different Tooltip placement options

<Tooltip position="left-start">
    <Button />
</Tooltip>
<Tooltip position="top">
    <Button />
</Tooltip>
<Tooltip position="right-end">
    <Button />
</Tooltip>
left-starttopright-end

Important info about some Tooltip specifics

#Disabled Elements

The Tooltip component is designed not to appear on any element with the CSS attribute pointer-events: none. You can override this by wrapping the child element in a <span>:

<Tooltip text="This tooltip doesn't work">
    <Button disabled />
</Tooltip>
<Tooltip text="This tooltip does">
    <span>
        <Button disabled />
    </span>
</Tooltip>
